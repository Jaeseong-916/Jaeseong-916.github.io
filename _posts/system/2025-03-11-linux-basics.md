리눅스마스터 1급을 준비하면서 정리한 리눅스의 기초 내용입니다.

## 1. 리눅스의 정의

리누스 토발즈에 의해 오픈소스로 개발된 OS입니다. 처음에는 리눅스 커널만을 의미했지만, 현재는 FSF(Free Software Foundation)가 GNU 프로젝트를 통해 OS의 형태로 제공하고 있습니다.

## 2. 리눅스의 일반적 특징

- **이식성(Portability)**
- **자유 소프트웨어** - GPL 또는 LGPL 라이선스
- **멀티 유저 / 멀티 프로그래밍**
- **계층적 파일 시스템**
- **셸(Shell)**
- **보안** - netfilter, iptables, SELinux 등

## 3. 리눅스의 기술적 특징

### 모놀리딕 커널 (Monolithic Kernel)
운영체제가 제공하는 모든 기능을 하나의 커널로 제공합니다. 동적 로드가 가능한 커널 모듈과 동적 설정을 위한 프로퍼티 기능도 지원합니다.

### 장치의 파일화
리눅스에서는 모든 장치를 파일로 취급합니다.

### 다양한 파일 시스템 지원
- Windows 파일 시스템 (FAT32, NTFS)
- 네트워크 파일 시스템 (SMB, CIFS)
- 저널링 파일 시스템 - 시스템 충돌 및 전원 문제로부터 복구 가능

### 가상 메모리
- **요구 페이징**: 사용 중인 메모리만 물리 메모리에 로드, 사용빈도가 낮은 메모리는 디스크에 저장

### 스왑
- 스왑 빈도 변경: `/etc/sysctl.conf`의 `vm.swappiness` 설정

### 동적/정적 라이브러리
- 로더는 `LD_LIBRARY_PATH`와 `/etc/ld.so.conf`를 검색하여 동적 라이브러리 로드
- `/etc/ld.so.conf` 변경시 `ldconfig` 명령어로 `ld.so.cache` 갱신 필요

### 가상 콘솔
- 다중 콘솔 생성: `Ctrl + Alt + F1 ~ F6`
- X 윈도우: `Ctrl + Alt + F7`

## 4. GNU와 오픈소스 라이선스

### 자유 소프트웨어의 4가지 자유

- **자유 0**: 어떠한 목적을 위해서도 실행할 수 있는 자유
- **자유 1**: 작동 원리를 연구하고 필요에 맞게 변경할 수 있는 자유
- **자유 2**: 복제 및 배포할 수 있는 자유
- **자유 3**: 프로그램을 향상하고 공동체에 환원할 수 있는 자유

### 주요 오픈소스 라이선스 비교

| 라이선스 | 소스코드 공개 의무 | 상업적 이용 | 2차 라이선스 |
|----------|-------------------|-------------|-------------|
| **GPL** | 수정 코드 모두 GPL로 공개 | 가능 | 불가 |
| **LGPL** | 수정 부분만 공개 (단순 활용 시 의무 없음) | 가능 | 가능 |
| **BSD** | 없음 | 가능 | 가능 |
| **Apache** | 없음 (변경사항 안내 필수) | 가능 | 가능 |
| **MIT** | 없음 | 가능 | 가능 |

## 5. 리눅스 배포판

- **슬랙웨어**: 가장 오래된 배포판
- **데비안**: Ubuntu, Linux Mint 등의 기반
- **레드햇**: RHEL, CentOS, Fedora, Rocky Linux 등의 기반
