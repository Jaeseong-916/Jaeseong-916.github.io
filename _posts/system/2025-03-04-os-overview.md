리눅스마스터 1급을 준비하면서 정리한 운영체제의 개요입니다.

## 1. 운영체제의 정의

1. 사용자와 하드웨어 간 상호작용을 위해 사용자 인터페이스를 제공하는 시스템 소프트웨어
2. 컴퓨터의 중앙처리장치, 메모리 및 주변 장치 등 하드웨어를 효율적으로 관리
3. 프로그램이 동작할 수 있는 환경, 프로그래밍 인터페이스 제공

## 2. 운영체제의 목적

1. **처리능력(Throughput)**의 향상
2. **반환 시간(Turnaround Time)**의 최소화
3. **신뢰도(Reliability)**의 향상
4. **사용 가능도(Availability)**의 향상

## 3. 운영체제의 역할

1. 하드웨어 제어, 입력, 출력 등을 관리
2. 하드웨어 자원 공유 수행
3. 자원 스케줄링 제공
4. 시스템 호출(system call) 제공
5. 오류 복구 기능 제공
6. 파일 관리 및 데이터베이스 기능 및 네트워크 기능 제공
7. 사용자 인터페이스 제공

## 4. 운영체제의 구조와 기능

### 구조

1. 시스템의 기능을 사용하기 위해 프로그램을 실행하거나 쉘(Shell), GUI, 배치 작업(batch job)을 통하여 기능 사용
2. 프로그램은 시스템 호출을 통하여 커널 서비스 이용
3. 운영체제는 다양한 하드웨어에 대한 이식성을 향상하기 위해 **하드웨어 추상 계층(HAL)** 제공

### 기능

- 리소스 관리, 자원 스케줄링
- 하드웨어 관리, 네트워크 제어
- 데이터 관리, 자원 공유, 자원 보호
- 오류 검사/복구, 가상화

## 5. 운영체제의 운용 기법

| 세대 | 운용 기법 |
|------|-----------|
| 1세대 | 일괄 처리 시스템 (Batch Processing) |
| 2세대 | 다중 프로그래밍 / 다중 처리 시스템 |
| 3세대 | 시분할 시스템 (Time Sharing) |
| 4세대 | 다중 모드 시스템 |
| 5세대 | 분산 처리 시스템 |

## 6. 운영체제의 사례

### 데스크톱 및 서버

- Windows, macOS
- **리눅스** - 리누스 토발즈가 개발, GNU를 통해 발전
- 유닉스

### 모바일 및 임베디드

- Android, iOS
- 타이젠(Tizen)
- 임베디드 리눅스 (라즈비안, webOS)

### IoT 운영체제

- Linux (Android Things, Ubuntu Core)
- RTOS (FreeRTOS, VxWorks, QNX)
- 경량 OS (Contiki, TinyOS, RIOT)
